{"id":"tool-snapshot-single-store-invariant","title":"Tool snapshot store has a single source of truth — never duplicate it in pi-fleet-extension","body":"## Invariant\n\nThe tool snapshot globalThis store lives in **one place only**: `packages/fleet-core/src/services/tool-registry/tool-snapshot.ts`. The MCP HTTP server in `admiral/_shared/mcp.ts` reads `getToolsForSession(token)` from that file. All registration paths MUST funnel into that single store via `FleetServices.mcp.registerTools(token, tools)`.\n\nDO NOT recreate a parallel `tool-snapshot.ts` inside `pi-fleet-extension/src/agent/provider-internal/` (or anywhere else). Doing so creates a separate globalThis store, which silently breaks ACP CLI tool exposure: the pi side registers tools that the MCP server cannot see.\n\n## The trap (round 4 incident)\n\nEarlier rounds delegated tool snapshot to a pi-local `provider-internal/tool-snapshot.ts` for \"self-contained adapter\" reasons. The result:\n- pi registered PI tools (Read/Write/Bash/etc.) into the pi-local store.\n- The MCP server read from the fleet-core store and saw an empty session — so ACP CLI received zero tools.\n- Type-check, build, and unit tests all passed because each store was internally consistent. The bug only manifested at runtime via missing tool exposure.\n\n## Correct registration flow\n\n`packages/pi-fleet-extension/src/agent/provider-internal/provider-stream.ts` MUST do the union registration:\n\n```ts\nfunction specToTool(spec: AgentToolSpec): Tool {\n  return {\n    name: spec.mcp?.exposeAs ?? spec.name,\n    description: spec.description,\n    parameters: spec.parameters,\n  };\n}\n\nconst fleetTools = getFleetRuntime().fleet.tools.map(specToTool);\nconst mcpTools = [...(piTools ?? []), ...fleetTools];\nmcpApi().registerTools(sessionToken, mcpTools);\n```\n\nBoth PI tools (host-provided) and `fleet.tools` (fleet-core auto-registered: sortie/squadron/taskforce/carrier_jobs) MUST be registered together so the ACP CLI sees the union.\n\n## How to apply\n\n- Future maintainers of `provider-internal/`: do NOT extract tool snapshot logic into a \"lightweight pi adapter\".
