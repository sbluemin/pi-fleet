{"id":"fleet-memory-prototype-implementation","title":"Fleet Memory prototype implementation summary","body":"# Fleet Memory prototype implementation summary\n\nThis page records the current prototype shape created during the Fleet Memory session.\n\n## Placement\n\nFleet Memory is implemented as an internal `extensions/fleet/memory/` module and is wired from the fleet extension startup path.\n\n## Workspace-local store model\n\nThe prototype uses a lazy-created `.fleet-memory/` workspace store with separate areas for:\n\n- `raw`: immutable captured source material.\n- `wiki`: approved markdown knowledge pages.\n- `schema`: memory doctrine and structural metadata.\n- `log`: append-only operational records and AARs.\n- `queue`: proposed memory patches awaiting approval or rejection.\n- `archive`: rejected or applied patch records and AAR audit copies.\n- `conflicts`: conflict records and future repair/audit state.\n- `index.json`: deterministic index metadata.\n\n## PI tools\n\nThe prototype exposes these Fleet Memory tools:\n\n- `memory_ingest`: capture raw source and propose a wiki patch.\n- `memory_briefing`: deterministic wiki briefing lookup by id, tag, title, or body.\n- `memory_aar_propose`: propose or append an AAR/log entry.\n- `memory_drydock`: run static integrity and safety inspection.\n- `memory_patch_queue`: list, show, approve, or reject queued patches.\n\n## Slash commands\n\nThe prototype exposes these command names:\n\n- `fleet:memory:status`\n- `fleet:memory:queue`\n- `fleet:memory:show`\n- `fleet:memory:approve`\n- `fleet:memory:reject`\n- `fleet:memory:drydock`\n\n## Behavioral contract\n\n- Ingest captures raw source separately from wiki pages.\n- Wiki changes are proposed as patches and are not merged until approval.\n- Rejected patches are archived without mutating wiki/log state.\n- AARs can be proposed, or explicitly auto-applied as append-only log entries.\n- Memory tools use unique prompt manifest tags to avoid registration collisions.\n- Store creation is lazy so normal startup/import does not create `.fleet-memory` by accident.\n\nraw_source_ref:
